# Правила чата Degradach

Одностраничный статический сайт с правилами чата **Degradach**.
Создан как замена telegra.ph, который заблокирован в РФ — чтобы работяги,
заходящие в чат, могли прочитать правила и не улетать в бан.

🌐 **URL:** [rules.degradach.ru](https://rules.degradach.ru)

---


Никаких сборщиков, фреймворков и npm-зависимостей. Просто статический HTML.

> выводится одно из четырёх слов — ЧИНАЗЕС / ЗУБЫ КОРЕГА / КРОССОВОК / КОНЬХ.
> Клик по этому невидимому тексту запускает соответствующий эффект.

## Деплой на GitHub Pages

1. Создайте публичный репозиторий (например, `degradach-rules`).
2. Залейте в корень три файла: `index.html`, `CNAME`, `README.md`.
3. В настройках репозитория: **Settings → Pages → Source → Deploy from branch → `main` / root**.
4. В течение 1–5 минут сайт поднимется на `https://<user>.github.io/degradach-rules/`.

### Подключение кастомного домена `rules.degradach.ru`

1. У DNS-провайдера `degradach.ru` добавьте запись:

   | Тип  | Имя   | Значение                  | TTL   |
   |------|-------|---------------------------|-------|
   | CNAME | rules | `<user>.github.io`.       | 3600  |

   ⚠️ В конце значения CNAME **обязательно** должна быть точка.

2. В настройках репозитория: **Settings → Pages → Custom domain** → впишите
   `rules.degradach.ru` → **Save**.
3. Поставьте галочку **Enforce HTTPS** (появится через 10–30 минут после
   выпуска сертификата Let's Encrypt).
4. Файл `CNAME` в репозитории уже лежит — GitHub автоматически подхватит домен
   при каждом деплое.

## Как обновить правила

Просто отредактируйте `index.html` — все правила лежат в нём же, в виде
семантических списков `<ol class="rules-list">`. Коммит → push → GitHub Pages
автоматически переопубликует.
